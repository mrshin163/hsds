# 이슈 레포트

결재할문서 - 결재할/완료 문서
* 특정 부서에 속한 사용자의 경우, 본인이 속한 부서의 품의서 모두 조회 가능하도록 수정


결재할문서 - 완료 문서
* 결재할문서 - 진행중 문서 추가에 따른 APPROVAL_STATUS_CD 조회 조건 수정
* APPROVAL_STATUS_CD IN ('NREJ, 'NAPR', 'EAPR')


결재할문서 - 진행중 문서 페이지 추가
* APPROVAL_STATUS_CD = 'NING'


최종 결재, 반려, 상신취소
* 품의서 APPROVAL_DATE = SYSDATE


정산서 팝업 ANCHOR 태그로 수정


결재할문서 - 결재할 문서
* 상신자부서, 상신자명 조회 조건 추가
* 상신자부서 LIKE 검색으로 수정


상신일 조회 조건 년, 월에서 캘린더로 변경


승인, 심포지움 요청취소, 심포지움 반려
* 일괄 처리 기능은 ajax 동기 처리


NULLABLE 컬럼에 NULL 업데이트 시, 마이바티스 오류 발생
* jdbcType=VARCHAR


정산서 - 카드사용내역
* 항목명 수정 : '공정거래규약' -> '공정경쟁규약'
* ACCOUNT_CD 로 구분하여 노출 항목 유동적으로 처리
* 1차 결재 예정자가 준법경영팀인 품의서만 '세부계정' 수정 허용
* N건의 카드사용내역 중, 아무 카드사용내역을 선택해서 수정하더라도 모두 동일한 '세부계정' 값을 가지도록 수정
* 승인 시, ACCOUNT_CD IN (3, 4) 인 경우만 '예산별도계정명' 입력하도록 노출
* ACCOUNT_CD = 5 인 경우만 '회의록' 노출
* ACCOUNT_CD IN (1, 2, 3, 4) 인 경우만 '제품군/명', '거래처' 노출


승인취소
* 최종 결재자의 차수가 1차인 경우, APPROVAL_STATUS_CD = 'NREQ'
* 심포지움 원천 품의서 APPROVAL_STATUS_CD = 'SING'


사용자 권한 관리
* 1명의 사용자는 N개의 권한을 가지도록 수정


상신취소, 재품의
* 대체 상신자도 해당 업무 가능하도록 수정


상신취소
* 심포지움의 원천 품의서 APPROVAL_STATUS_CD = 'SR'


재품의
* 품의서, 품의내역 외에 첨부파일도 복사
* 재품의 상신이 완료 되면, 반려된 이전 품의서는 삭제 처리 : 유대웅 주임에게 프로세스 전달 완료


정산서 - 카드사용내역 - 첨부파일 팝업
* '물리파일명.확장자' 로 표기하도록 수정


첨부파일 업로드
* 이미지 파일은 리사이징 및 포맷 컨버팅 처리 (세로 700px 기준)
* 리사이즈 및 포맷 컨버팅 라이브러리 : Apache Tika http://tika.apache.org/


DB 스키마에 맞춰 INPUT TYPE='TEXT' MAXLENGTH 조정


반려 및 심포지움 반려 메일 발송
* 메일 발송 실패 익셉션 제거 : 수신자 메일 주소가 존재하지 않는 경우, 업무 로직이 롤백 되어 익셉션 throw 제거
* 운영도 4/1 부터 정상 발송 예정 (현재, 이청일 대리에게 발송 처리) : ApprovalService.java restore(), ApprovalRqService.java restoreSymposium() 수정


진행중, 완료, 반려, 결재할 - 결재할, 결재할 - 결재할 (관리자용), 결재할 - 진행중, 결재할 - 완료 리스트 페이지
* 총 7개 리스트에 SQL 조회 조건 변경 : '상신일' -> '카드사용일'


결재할 - 결재할 문서
* 부서코드, 사번 검색 추가 (부서코드와 상신자부서 / 사번과 상신자명 AND 조건)
