1. 심포지움 결재요청 취소 
	:: 일반 영업사원이 결재요청한 심포지움 결재요청을 회수하는 경우
	:: 심포지움 관리자가 결재요청건을 거부하는 경우
	
	--> 둘 다 SR건의 삭제로 진행한다.
	    이 때 함께 올라온 첨부파일도 삭제한다.
	    
	1.1 품의서 삭제
	1.2 품의 내역 삭제
	1.3 품의 결재라인 (해당없음)
	1.4 품의 첨부파일 삭제
	1.5 첨부파일 상세 삭제 
	1.6 첨푸파일을 NAS에서 파일을 실제 삭제
	1.7 첨부마스터 삭제
	
	:: 취소(거부)된 카드승인 건은 결재미상신에서 다시 조회됨
	
	
	
2. 품의서 클린징
	:: 결재 미상신에서 품의서 작성을 들어가면 품의서가 임시(T)로 생성된다.
	   상신전에 올리는 품의서 첨부파일을 첨부파일 테이블에 기록되지만,
	   상신해야 의미가 있어진다.
	   따라서, 상신하지 않고 화면을 떠나면 임시(T)상태의 품의서가 남는데
	   이는 클린징 대상이다. 
	   주기적으로 이를 제거해주도록 한다.
	   
	   1.1 품의서 삭제 (EAC_APPROVAL.STATUS_CD가 'T'이고 생성일이 지난달인것)
	   1.2 품의서 내역 삭제
	   1.3 품의서 결재라인 삭제
	   1.4 품의서 첨부파일 삭제
	   1.5 첨부파일 상세 삭제
	   1.6 첨부파일을 NAS에서 파일을 실제 삭제
	   1.7 첨부파일 마스터 삭제
	   
	   
3. 첨부파일 입장에서 첨부파일 클린징
	:: 첨부파일에는 관련 업무키(BIZ_KEY)가 있어 어떤 업무에서 올라온것인지에 대한 
	   연결고리가 있다. 이 연결고리가 없는 첨부파일은 존재 가치가 없는 것이므로 삭제 대상이다.
	   
	   1. 첨부파일 마스터의 BIZ_TP_CD 의 업무에서 BIZ_KEY가 없는 것을 추출한다.
	      첨부파일 마스터 기준으로 아웃터 조인하여 얻는다. 
	      BIZ_KEY는 멀티키이고 구분자가 |이다. 
	      
	   2. 첨부파일 마스터를 지운다.
	   
	   3. 첨부파일 상세를 지운다.
	   
	   4. 첨부파일을 NAS에서 파일을 실제 삭제
